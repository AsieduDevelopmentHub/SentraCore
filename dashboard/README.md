# SentraCore Dashboard (Flutter Desktop)

The **SentraCore Dashboard** is the desktop visualization and interaction layer of the SentraCore platform.

Built with **Flutter for Windows**, the dashboard connects to the local intelligence engine to deliver **real-time telemetry visibility**, **system health analytics**, and **explainable operational insights** through an interactive desktop experience.

The dashboard is designed to provide a low-latency interface for observing system behavior while exposing actionable intelligence generated by the backend engine.

---

## Overview

The dashboard communicates with the local SentraCore engine through a combination of:

- **REST APIs** for state retrieval and command operations
- **WebSocket streams** for real-time updates and event delivery

This architecture separates intelligence processing from presentation while maintaining responsive desktop performance.

## Connectivity

By default, the dashboard connects to a locally running SentraCore engine.

### REST API

```text
http://127.0.0.1:8740/api/v1/
```

Used for:

- System status
- Health queries
- Process inspection
- Historical retrieval
- Control actions

---

### WebSocket

```text
ws://127.0.0.1:8740/ws/live
```

Used for:

- Live telemetry streaming
- Event propagation
- Dashboard updates
- Realtime intelligence output

---

## Configuration

If the engine host or port changes, update the dashboard connection settings.

Current location:

```text
dashboard/lib/services/engine_service.dart
```

Recommended future enhancement:

```text
Settings → Engine Endpoint Configuration
```

This would allow runtime endpoint configuration without rebuilding the application.

## Local Development

### 1. Start SentraCore Engine

From the repository root:

```powershell
.venv\Scripts\python -m engine.main
```

Expected startup:

```text
Engine available at:
http://127.0.0.1:8740
```

---

### 2. Start Dashboard

Open a second terminal.

```powershell
cd dashboard

flutter pub get

flutter run -d windows
```

The dashboard should automatically connect to the local engine.

---

## Build Desktop Application

Generate a Windows desktop build:

```powershell
cd dashboard

flutter build windows
```

Build output:

```text
dashboard/build/windows/x64/runner/Release/
```

## Troubleshooting

| Issue | Possible Cause |
|---|---|
| Dashboard cannot connect | Engine not running |
| Empty realtime panels | WebSocket unavailable |
| REST requests fail | Incorrect engine endpoint |
| Flutter build fails | Missing Windows desktop support |
| UI not updating | Hot reload required |
