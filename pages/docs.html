Documentation

Practical notes for installing SentraCore from releases, running locally, and hitting core API endpoints. For deeper detail, see the repository README and source.

Installation (from releases)

  1. Visit GitHub Releases and open the latest tag.
  2. Download the artifact for your OS (Windows installer, macOS zip, Linux AppImage).
  3. Windows: run the installer; shortcuts place the dashboard and engine under the chosen install path.
  4. macOS: unzip, move the .app to Applications if you like, and open it—Gatekeeper may prompt on first launch for unsigned/ad-hoc builds.
  5. Linux: chmod +x the AppImage, then execute it from a terminal or file manager.

System requirements (typical)

  • Windows: 64-bit Windows 10 or newer recommended.
  • macOS: recent macOS on Apple Silicon or Intel (CI builds target arm64).
  • Linux: x86_64 glibc-based distro with FUSE/libfuse as needed for AppImage.
  • Network: localhost HTTP between dashboard and engine unless you configure otherwise.

Running from source (developers)

Clone the repository, create a Python virtual environment, install requirements.txt, and run the engine entrypoint; use Flutter tooling inside dashboard/ for UI development. See the repo README for exact commands.

API overview

The engine exposes HTTP endpoints for health and API versioning. Example base URL when running locally: http://127.0.0.1:<port> (port comes from your engine config).

Method Path Purpose
GET /api/v1/health Liveness / readiness check for the engine.

Additional routes live under /api/v1/ as implemented in the codebase—consult the FastAPI app for the full list.
